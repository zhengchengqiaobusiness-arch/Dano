# Domain Docs

This repo uses a multi-context domain-doc layout.

Before exploring, read `CONTEXT-MAP.md`, then the relevant `CONTEXT.md` files it points to, such as root `CONTEXT.md` and `deploy/CONTEXT.md`.

Read `docs/adr/` for architectural decisions that touch the area being changed.

When output names a domain concept, use the glossary vocabulary from the relevant `CONTEXT.md`. If output contradicts an ADR, surface that conflict explicitly.
